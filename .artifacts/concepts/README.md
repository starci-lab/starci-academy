# concepts/ — định hướng feature (nguồn brainstorm thay web)

Mỗi file = 1 concept ĐỊNH HƯỚNG cho 1 feature/flow: ý đồ, người dùng, outcome,
nguyên tắc — **KHÔNG phải spec UI** (spec sống ở `proposals/` sau khi brainstorm chốt).

Brainstorm skills ground từ ĐÂY + `states/` — **KHÔNG search web**.
Thiếu concept / concept mơ hồ → DỪNG hỏi thầy, không tự chế.

## Quy ước

- Tên file: `<feature-slug>.md` (kebab-case, trùng slug với `proposals/` nếu có).
- Tạo mới: copy `_TEMPLATE.md`, điền đủ các mục — mục nào chưa chốt ghi rõ `❓ chờ thầy`.
- Cập nhật `INDEX.md` mỗi khi thêm/đổi trạng thái concept.
