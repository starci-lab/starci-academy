Naming convention — cách trò sẽ làm

Full English · Family số nhiều · giữ pattern Component · descriptor. Scan ~130 story.

1. Cấu trúc title path

Tier/ Family/ Component/ Variant
  • Tier-1 (5 cố định): Primitives · Design · Block · Layouts · Overlays
  • Family = số NHIỀU cho họ component-type (Cards·Buttons·Chips·Lists·Forms·Stats·Texts); domain-noun giữ English (Learn·Feed·Commerce·Grading…)
  • Component / Variant = PascalCase English (giữ nguyên — đã ổn)

Ví dụ: Design/Cards/ContinueCard/Plain · Primitives/Buttons/Button · Layouts/CourseContents

3. Display name = kiểu COMPONENT-IDENTIFIER full english

Không prose, không ·, không tiếng Việt. Viết như tên component:
  • Standalone = PascalCase: NoNeedToShow · TitleOnly · WithProgress
  • Compound / variant = dot notation Parent.Variant (thay ·): Skeleton.ABC · Button.Primary · DropdownItem.Delete
  • Descriptor khớp prop/state code (Primary·Loading·Empty·Skeleton), PascalCase từng từ

Trùng khớp name: ≈ export const (đã PascalCase) → nhiều chỗ chỉ cần bỏ name: prose, để Storybook tự lấy tên export.
